C++各版本新特性

2026-04-14
C++
6.2k words

C++11：

一、泛型编程和编译器特性：

让编译器在编译阶段完成更多计算，并大幅降低模板报错的晦涩程度。

  • auto 类型推导： 让编译器通过初始化表达式推断变量类型。
  • decltype： 获取表达式或实体的静态类型，常用于泛型编程中。
  • 尾随返回类型 (Trailing Return Type)： 结合 auto 和 decltype，解决模板函数返回值依赖参数类型的问题（如 auto add(T1 a, T2 b) -> decltype(a + b)）。
  • 可变参数模板 (Variadic Templates)： 允许模板接受任意数量的参数（template<typename... Args>），极大地增强了模板的灵活性。
  • constexpr 常量表达式： 声明变量或函数可以在编译期求值，将运行期计算提前到编译期。
  • 类型别名 (using 语法)： 替代 typedef，支持模板别名（如 template<typename T> using MyVec = std::vector<T>;）。
  • 静态断言 (static_assert)： 编译期的断言检查，条件为假时立刻终止编译并输出自定义错误信息。
  • 外部模板 (extern template)： 显式阻止编译器在当前翻译单元实例化某个模板，减少编译时间和目标文件体积。
  • 默认模板参数增强： 允许为函数模板提供默认参数（C++03 仅支持类模板）。
  • 局部和匿名类型作为模板参数： C++11 终于允许将函数内部定义的结构体或未命名的类型传入模板。

二、移动语义和右值引用：

1、C++11引入右值引用由 && 表示。

2、std::move：将左值强制转换为右值引用，触发移动语义（只是类型转换，不实际移动数据）。

3、std::forward：完美转发，保留参数的左值 / 右值属性，直接透传（核心用于泛型函数，如模板工厂函数）。

4、移动构造 / 移动赋值函数：类的成员函数，实现对象所有权的直接转移（替代深拷贝，大幅提升性能）。（可以通过禁用拷贝构造，拷贝赋值做到 move-only 语义)

三、智能指针和内存管理：

  • 智能指针：把内存管理从裸指针封装成对象，指针和对象周期绑定，随对象析构内存释放，指针也相应销毁，防止内存泄漏。定义管理内存对象的所有权。
  • 智能指针 (<memory>)：
    • std::unique_ptr：独占所有权，取代被废弃的 std::auto_ptr。
    • std::shared_ptr：共享所有权，基于引用计数（自带线程安全的计数器）。
    • std::weak_ptr：弱引用，解决 shared_ptr 的循环引用引起的内存泄漏问题。
    • std::make_shared：更安全、更高效的分配共享指针的方式

四、并发与多线程原生支持（C++11 首次引入标准并发库）

  • 标准线程库 (std::thread)： 引入了跨平台的多线程支持。
  • 同步原语： 增加了互斥锁 (std::mutex)、条件变量 (std::condition_variable) 和锁管理器（如 std::lock_guard、std::unique_lock，同样基于 RAII）。
  • 异步编程 (std::future & std::promise)： 提供了获取异步任务结果的机制。
  • 原子操作 (std::atomic)： 引入了无锁编程的基础，并正式定义了 C++ 的内存模型（Memory Model）。

五：初始化、语法糖与控制流（提升可读性）

让 C++ 的代码写起来更现代、更符合人类直觉。

  • Lambda 表达式： 引入闭包（Closure），支持就地定义匿名函数，可捕获外部作用域的变量，极大简化了 STL 算法库的使用。
  • 统一初始化 (Uniform Initialization)： 使用大括号 {} 初始化任何对象，并严格禁止窄化转换（Narrowing Conversion，例如 int x {1.2}; 会报错）。
  • 初始化列表 (std::initializer_list)： 允许使用 {1, 2, 3} 这样的语法来初始化标准容器或自定义类。
  • 基于范围的 for 循环 (Range-based for loop)： 如 for (auto& item : collection)。
  • nullptr： 强类型的空指针常量（类型为 std::nullptr_t），彻底替代宏 NULL 和整型 0。
  • 强类型枚举 (enum class)： 枚举值不会隐式转换为整型，且不污染外部命名空间。
  • 原生字符串字面量 (Raw String Literals)： 以 R"(...)" 格式表示，忽略转义字符，极其适合写正则表达式、JSON 或文件路径。
  • 用户自定义字面量 (User-defined Literals)： 允许自定义后缀（如 1200_km 或 3.14_rad），并重载相应的运算符进行解析。

六：面向对象与类设计（精细化控制）

赋予了类作者对继承体系和默认行为绝对的控制权。

  • override： 显式标记重写基类的虚函数，由编译器检查签名是否完全一致，防止拼写错误。
  • final： 标记类不可被继承，或虚函数不可被再次重写。
  • = default： 强制编译器生成默认的特殊成员函数（如默认构造、拷贝构造、析构函数）。
  • = delete： 显式禁用某个函数（可用于禁用拷贝构造，或禁用特定类型的隐式转换重载）。
  • 委托构造函数 (Delegating Constructors)： 同一个类中，一个构造函数可以在初始化列表中调用另一个构造函数。
  • 继承构造函数 (Inheriting Constructors)： 使用 using Base::Base; 将基类的构造函数直接引入派生类，避免写样板代码。
  • 类内成员初始化 (In-class Member Initializers)： 可以在类定义时直接为非静态成员变量赋初值。

七：底层机制与内存模型（安全性与极限性能）

  • 并发内存模型 (Memory Model)： 正式在标准层面定义了多线程下的内存可见性和执行顺序，为无锁编程奠定基础。
  • noexcept 关键字： 替代了废弃的动态异常规范（throw()），明确告诉编译器函数不会抛出异常，有助于编译器进行极致的优化（特别是对移动构造函数尤为重要）。
  • 内存对齐控制 (alignas 和 alignof)： 提供了标准化的方式来查询和指定数据结构的内存对齐方式。
  • 无限期联合体 (Unrestricted Unions)： 取消了以往“联合体成员不能包含非平凡构造/析构函数对象”的限制（例如可以在 union 里放 std::string 了）。
  • 基础类型扩充： 正式引入 long long 至少 64 位整数类型，以及用于 Unicode 的 char16_t 和 char32_t。
  • 属性 (Attributes) 语法： 引入标准化的属性标记，如 [[noreturn]]（告诉编译器该函数永远不会正常返回，比如死循环或直接抛出异常）。

八：标准库的全面扩充（STL 现代化）

除了前文提到的智能指针，标准库增加了一大批现代化组件。

  • 多线程库 (<thread>, <mutex>, <condition_variable>, <future>, <atomic>)： 提供原生多线程、同步原语、异步操作及原子类型支持。
  • 无序容器 (Unordered Containers)： 基于哈希表实现的 std::unordered_set、std::unordered_map 及其 multi 版本。
  • 定长数组 (std::array)： 封装了原生 C 数组的薄层，既有原生数组的性能，又有标准容器的接口（如 .size() 和迭代器）。
  • 单向链表 (std::forward_list)： 对标 C 语言中的单链表，提供比 std::list 更低的内存开销。
  • 元组 (std::tuple)： 允许将多个任意类型的元素打包在一起。
  • 正则表达式库 (<regex>)： 原生支持正则表达式的匹配、搜索和替换。
  • 高精度时间库 (<chrono>)： 基于时钟 (Clock)、时长 (Duration) 和时间点 (Time point) 的现代时间处理库。
  • 随机数库 (<random>)： 淘汰了简单的 rand()，引入了多种高质量的随机数生成引擎（如梅森旋转算法 std::mt19937）和丰富的分布（如正态分布、均匀分布）。
  • 多态函数包装器与绑定 (std::function & std::bind)： 提供通用的回调机制。 统一了普通函数、函数指针、Lambda 以及成员函数的存储和调用方式。
  • 类型萃取 (<type_traits>)： 提供了一组模板元编程工具，用于在编译期查询、修改和判断类型的特征（如 std::is_integral<T>）。
  • 字符串转换工具： 引入了 std::to_string() 以及 std::stoi, std::stod 等极其方便的数字与字符串互转函数。

C++14：

一：所有权与资源管理（完美补齐）

  • 新增：std::make_unique<T>： C++11 只有 std::make_shared，导致在 C++11 中创建 unique_ptr 必须显式使用 new，这不仅繁琐，而且在作为函数参数传递时存在轻微的内存泄漏风险。C++14 补齐了 std::make_unique，自此 C++ 代码中可以彻底告别原生的 new 和 delete。

二：泛型编程与编译期特性（大幅解绑限制）

C++14 让模板和编译期编程变得更加自然，移除了 C++11 中很多严苛的限制。

  • 修改/增强：放宽的 constexpr 函数限制： C++11 的 constexpr 函数要求极严，函数体几乎只能有一条 return 语句。C++14 大幅解绑，允许在 constexpr 函数内部使用局部变量、if/else 条件分支、switch 语句以及 for/while 循环，让编译期编程终于像写普通代码一样顺手。
  • 修改/增强：普通函数的返回值类型推导： 不再仅限于 Lambda，普通函数也可以直接用 auto 作为返回值类型，编译器会自动根据 return 语句推导（无需再写 C++11 麻烦的尾随返回类型 -> decltype(...)）。
  • 新增：decltype(auto)： 结合了 auto 的推导能力和 decltype 保留引用/cv限定符（const/volatile）的能力。主要用于泛型编程中完美转发函数的返回值。
  • 新增：变量模板 (Variable Templates)： C++ 以前只有类模板和函数模板，C++14 允许定义变量模板。例如定义一个通用的圆周率：template<typename T> constexpr T pi = T(3.1415926535897932385);，你可以按需实例化为 float 或 double。

三：面向对象与类设计

  • 修改：聚合类（Aggregate）初始化的放宽： C++11 中如果一个类使用了“类内成员初始化”（比如 int x = 0;），它就不再是聚合类，也就不能用大括号 {} 按成员初始化。C++14 修复了这个问题，允许包含类内初始值设定的类继续使用聚合初始化。

四：初始化、语法糖与控制流（Lambda 再进化）

Lambda 表达式在 C++14 中得到了史诗级的增强，语法糖也让代码更具可读性。

  • 新增：泛型 Lambda (Generic Lambdas)： 允许在 Lambda 的参数列表里直接使用 auto（如 [](auto x, auto y) { return x + y; }）。这本质上是让编译器在底层自动生成了一个模板调用运算符（operator()）。
  • 新增：Lambda 捕获列表初始化 (Init-capture)（广义 lambda 捕获）： 这是极其重要的特性！C++11 的 Lambda 只能按值或按引用捕获外部变量，无法捕获只支持移动的对象（比如 std::unique_ptr）。C++14 允许在捕获列表中创建新变量并初始化，例如 [ptr = std::move(my_ptr)] { ... }，完美解决了 Lambda 与移动语义的结合问题。
  • 新增：二进制字面量： 可以直接使用 0b 或 0B 前缀书写二进制数（如 int mask = 0b101010;）。
  • 新增：数字分隔符 (Digit Separators)： 允许在数字常量中使用单引号 ' 作为分隔符，增强长数字的可读性（如 long budget = 1'000'000'000;）。

五：底层机制与内存模型

  • 新增：大小释放 (Sized Deallocation)： 允许重载带有大小参数的 delete 运算符（void operator delete(void* ptr, std::size_t size)）。在释放内存时，如果内存分配器知道要释放的具体字节数，可以极大地优化内存回收的性能。
  • 新增：[[deprecated]] 属性标记： 提供了一种标准化的方法来标记类、变量、函数或特定的命名空间已废弃。当其他开发者使用被标记的代码时，编译器会统一抛出警告。

六：标准库的全面扩充（实用工具集）

C++14 的标准库增加了一些非常实用的小工具和并发组件。

  • 新增：读写锁 (std::shared_timed_mutex 和 std::shared_lock)： 补齐了多线程编程中经典的“读者-写者”锁机制。允许多个线程同时读，但写线程独占。
  • 新增：标准用户自定义字面量： 标准库利用 C++11 的字面量特性，内置了一批方便的后缀。
    • 字符串："hello"s（直接生成 std::string 对象，而不是 C 风格指针）。
    • 时间：100ms, 2h, 5s（直接生成 <chrono> 库对应的时间段对象）。
  • 新增：std::exchange： 将一个新值赋给对象，并返回对象原来的旧值。在实现移动构造函数或重置状态时极其好用。
  • 修改/增强：Tuple 通过类型获取元素 (std::get<T>)： C++11 只能通过索引（std::get<0>(t)）获取元组元素，C++14 允许在类型唯一的情况下直接用类型获取（std::get<int>(t)）。
  • 新增：std::integer_sequence 和 std::index_sequence： 编译期的整数序列，这是模板元编程的重磅工具，极大简化了把 std::tuple 展开作为函数参数调用的复杂逻辑。
  • 新增：std::quoted： 用于 I/O 流的输入输出操作，能够自动处理包含空格的字符串的引号添加与解析。

C++17：

1、C++17引入正式的内存池实现：。这套机制被称为 PMR (Polymorphic Memory Resources，多态内存资源)。

2、std::string_view

3、std::optional

4、inline 变量：如果没有inline关键字会让多个 .cpp include 这个头文件会导致重复定义链接错误

C++20：

1、协程 。promise_type协程结构体。Promise 协议和 Awaiter 协议。co await，co return，co yield协程语义。awaiter三件套，await_ready()，await_suspend()，await_resume()。

2、C++20关于 std::atomic 的升级，等待机制 std::atomic::wait。
